## 4. 🟢 우선순위 4 — 슬라이드 2~3 "hook" 페르소나 표기 (선택)

### 4-1. 배경

사용자 요청 방향: 대시보드는 **다거래소 탭** (Binance / Bybit / OKX / ...) 로 바뀔 가능성 있음 (§5 참조). 이 경우:
- 바이낸스 탭 → 0.17% 임계값 (VIP0+BNB 0.15% + 0.02% buffer)
- 바이비트 탭 → 0.12% 임계값 (USDC-promo 0.10% + 0.02% buffer)
- 기타 거래소 → 각자 수수료 + buffer
- **공개**: "실제 live API 는 Binance 1개. 나머지는 각 거래소 수수료 엔벨로프 기반 시뮬"

### 4-2. 덱 반영 필요성

- 슬라이드 2 (Problem) — 수수료 장벽 설명. 거래소별 수수료 편차 언급하면 **탭 디자인의 복선**.
- 슬라이드 6 (Pricing) — UCB1 가격 공식 덱은 "premium-rate state" 서술. **거래소별 fee_base 가 다르다**는 뉘앙스 추가 가능.
- 슬라이드 10 (Shipping) — 라이브 범위 명시: `Binance live feed · Bybit/OKX fee-envelope simulation`.

**결정 기준**: §5 탭 디자인을 대시보드에 실제 구현하면 덱도 동기 필요. 시간 없으면 **생략**하고 대시보드만.

## 5. 🟢 우선순위 5 (신규 디자인) — 다거래소 임계값 탭 (대시보드 + 덱 동기)

### 5-1. 콘셉트

대시보드 상단에 거래소 탭:
```
[ Binance (live) ] [ Bybit ] [ OKX ] [ Upbit ] [ ... ]
```
각 탭 클릭 시:
- **임계값 라벨**: 해당 거래소의 수수료 + 0.02% buffer (예: Bybit 0.10% + 0.02% = **0.12%**)
- **거래 내역**: 해당 임계값 넘긴 기회만 표기 (실제 signal stream 은 Binance 기반이나 filter)
- **Live 뱃지**: Binance 만 `LIVE · real REST`, 나머지 `SIM · fee-envelope`

### 5-2. 공개 문구 (필수)

탭 헤더 또는 페이지 하단 고정 캡션:
> **Honesty note**: Due to time constraints, only Binance REST API is wired live. Bybit/OKX/Upbit tabs reproduce the same signal stream filtered through their public fee schedule (maker/taker/VIP0 USDC-promo) and realistic per-exchange slippage envelopes.
> Fee sources: Binance VIP0+BNB 0.075%×2=0.15%, Bybit VIP0 USDC-promo 0.05%×2=0.10%, OKX VIP0 0.08%×2=0.16%. Live migration = extend `FeeSource` adapter only.
