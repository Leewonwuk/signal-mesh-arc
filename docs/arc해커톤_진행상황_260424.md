## 2. 오늘 내린 주요 아키텍처 결정

### 2.1 Settle 버그 ≠ Circle 문제 — Arc testnet mempool 포화

**발견**: Arc RPC가 `txpool is full` (code -32003)로 raw EOA 제출을 거부.
**우회**: Circle Developer API `/v1/w3s/developer/transactions/transfer` 경로는 정상 → 60 tx 배치 완전 성공.
**의미**:
- Demo의 `consumers/executor_agent/main.py` (raw EOA + ethers) 경로는 **영상용 마켓 다이내믹스 (paper)**만 담당
- 실제 60+ on-chain tx 증빙은 **`scripts/circle_batch_settle.js`**가 담당
- 이 dual-path는 **SUBMISSION.md §5.x**에 정직하게 기재 완료

### 2.2 SCA 지갑은 Export Private Key 불가

처음엔 런북에 `Circle Console → Export private key`로 적혀있었으나, Circle Developer-Controlled Wallet (SCA type)은 보안 설계상 export 불가능. 우회:
1. 새 EOA 로컬 생성 (`eth_account.Account.create()`)
2. `faucet.circle.com`에서 새 EOA 주소로 바로 drip → USDC + native gas 확보
3. `.env`에 `EXECUTOR_PRIVATE_KEY`로 저장
4. 4 SCA 지갑은 Circle API로만 사용 (배치 스크립트)

이 내용은 **`docs/PRODUCT_FEEDBACK.md` Point 4**에도 자연스럽게 반영되어 있음 (판단 보너스 +$500 대상).

### 2.3 대시보드 레이아웃 재배치 (Allocator 먼저)

이전 순서: Stats → FeeExplorer (~420px) → Allocator (스크롤 필요)
**변경 후**: Stats → **Allocator section (hero)** → FeeExplorer → columns
**이유**: 1080p 녹화 1st viewport fold에 Allocator가 잡히도록 (Kent Beck + 비주얼 리뷰 공통 지적).

## 8. 주의사항 / Gotchas

- **Discord 봇 토큰**: `.env`의 `DISCORD_BOT_TOKEN`은 커밋 금지 (이미 `.gitignore` 처리됨)
- **Circle Entity Secret 회전**: 이미 `.env`에 저장된 값이 Circle Console에 등록된 것과 일치해야 함. Reset 금지 (회전 시 전 tx 서명 불가)
- **4지갑 SCA 주소는 고정**: `circle_balances.js`와 `circle_batch_settle.js`에 하드코딩됨 — 다른 Wallet Set으로 전환 시 둘 다 갱신
- **Arc testnet USDC는 18-dec native + 6-dec ERC-20 혼재**: `0x3600000000000000000000000000000000000000` 조회 시 6-dec. 실수하면 10^12× 오차 (PRODUCT_FEEDBACK.md Point 3 참조)
- **FeeExplorer 초기 POST 제거됨**: 대시보드 열기 전에 curl로 persona 세팅 필수
- **`scripts/circle_batch_settle.js:27` `AMOUNT='0.01'` hardcoded**: 영상 속에서 화면에 띄우면 안 됨 (Karpathy 경고)
- **배치 실행 시간**: 60 tx × 5초 평균 = ~5분. `--rate 3 --count 60` 기준
