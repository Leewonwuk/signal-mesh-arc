## Long description

```
AlphaLoop is an agent-to-agent marketplace on Arc where four specialist trading agents pay each other sub-cent USDC for arbitrage signals — amounts varying $0.0005 to the $0.010 per-action cap, sampled from the signal-quality distribution the executor's pricing policy emits. Arc's USDC-as-gas closes the loop without a human refiller.

What makes us different:

1. LIVE PRODUCTION BACKING. The dual-quote producer replays 1-second ticks from my v1.3 arb bot running on EC2 right now — 9 coins, pool ≈ $1,977 USDT, threshold 0.17%, running today. This hackathon demo is layer 2 on a real trading system, not a demo built to win a hackathon. No other Track 2 entry has a live revenue system behind the wheel.

2. ERC-8004 ON-CHAIN AGENT IDENTITY. Our AlphaLoopAgentRegistry contract at 0xb276b96f2da05c46b60d4b38e9beaf7d3355b7ab on Arc testnet emitted 5 AgentRegistered events — one per agent (3 producers · 1 meta · 1 executor), each containing a sha256 contentHash of the served agent card JSON. Judges fetch the card from https://signal-mesh.vercel.app/.well-known/agent-card/<role>, recompute sha256, and compare to the on-chain event payload. Content-addressed off-chain documents, cryptographically linked to on-chain events. Full details in docs/evidence/erc8004_registry.json.

3. LEARNED ALLOCATOR WITH A RECEIPT. A tabular Q-learner picks which of three trading strategies gets to trade. Round 1 it lost to a one-line if-statement — a Sutton-school adversarial review caught a reward-hacking bug. Round 2 ties the empirical optimum on a 47-tick walk-forward hold-out (p=0.49 vs ALL_V2, p=0.012 vs DIVERSIFY). Fail → diagnose → fix → receipt, all reproducible from committed data (docs/SUBMISSION.md §11.b).

4. REAL VARIABLE PRICING ON-CHAIN + MERKLE-ROOTED AUDIT. 150 settlements per demo run land on Arc testnet via Circle's Developer API, each with a different USDC amount sampled per signal-quality tier (60% low / 30% mid / 10% high-confidence at cap). Records (hash + amount, tab-separated) in docs/evidence/batch_tx_hashes.txt, Merkle root (sorted-pair SHA-256) in docs/evidence/merkle_root.txt. Run `make verify` and the entire evidence manifest is tamper-evident. No flat-$0.01 cron receipt.

5. HONEST DUAL-PATH. The EIP-3009 self-signing path ships in consumers/executor_agent/main.py. Arc testnet mempool was returning "txpool is full" during the build week so we pivoted the evidence burst to Circle's Developer API. SUBMISSION §5 names this tradeoff + three concrete custody failure modes (entity-secret leak, wallet freeze, API rate-limit-as-censorship). We'd rather judges see the audit than have the adversarial reviewer find it.

Stack: Arc testnet · USDC-as-gas · Circle Wallets (SCA) · Circle Developer API · x402 paywall wired · ERC-8004 identity registry · Gemini 2.5 Flash (with deterministic stub fallback). MIT-licensed.

Track: 🤖 Agent-to-Agent Payment Loop (primary) · 🪙 Per-API Monetization (secondary).

Judge shortcuts:
  • https://signal-mesh.vercel.app                          — dashboard
  • https://signal-mesh.vercel.app/scoreboard.html          — one-page judge-ready verification
  • https://testnet.arcscan.app/address/0xb276b96f2da05c46b60d4b38e9beaf7d3355b7ab  — registry contract
  • https://github.com/Leewonwuk/signal-mesh-arc/blob/main/docs/evidence/batch_tx_hashes.txt  — 150 tx manifest
  • `make verify`                                           — recompute Merkle root, 60-sec check
```
