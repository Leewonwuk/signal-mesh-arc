### A-2. Hero ribbon: 4개 → 6개 스탯 (`App.tsx:82-99`)

**현재**: raw / premium / on-chain tx / producers tracked — 4개 stat card.

**변경 후: 6개 stat card** (2줄로 자동 wrap, grid `repeat(auto-fit, minmax(170px, 1fr))`):

| 슬롯 | 라벨 | 값 바인딩 | 색상 클래스 | 근거 |
|---|---|---|---|---|
| 1 | `raw signals` | `health?.signals.raw` | cyan | 기존 유지 |
| 2 | `premium signals` | `health?.signals.premium` | purple | 기존 유지 |
| 3 | `on-chain tx` | `health?.tx_settled` | green | 기존 유지 |
| 4 | `producers tracked` | `producers.length` | amber | 기존 유지 |
| 5 ⭐ | `net PnL (paper)` | `tx.reduce((s,t)=>s+t.amount,0)` 임시 · 또는 `/economics/summary` 신규 엔드포인트 | `emerald` (기존 색 없으면 green 재사용) | §4 "closed outcome loop" 증거를 정면에 |
| 6 ⭐ | `regime` | `premium[premium.length-1]?.regime ?? '—'` | `indigo` (없으면 purple) | §7 GBM regime classifier가 실제 작동 중임을 시각화 |

**구현 스케치 (`App.tsx`)**:
```tsx
const netPnlUsd = tx.reduce((acc, t) => acc + t.amount, 0) // 임시: 정산 합계
const latestRegime = premium.length > 0
  ? premium[premium.length - 1].regime ?? '—'
  : '—'

// grid 내부 추가:
<div className="stat">
  <div className="label">net settlement $</div>
  <div className="value emerald">${netPnlUsd.toFixed(3)}</div>
</div>
<div className="stat">
  <div className="label">regime</div>
  <div className="value indigo">{latestRegime}</div>
</div>
```

**이유**:
- **NetPnL**: 심사관 눈높이 "이게 돈 벌었나?"의 1초 답변. `tx.amount`는 정산 금액 합이라 PnL과 정확히 같진 않지만, 제출 하루 전에 새 엔드포인트 만들기보다 표시 라벨을 `net settlement $` 로 정직하게.
- **Regime**: §7의 GBM classifier가 장식이 아님을 1초에 입증. `premium[last].regime`으로 기존 payload에서 뽑음 — 신규 fetch 0개.

**Option A+ (신규 엔드포인트 허용 시)**:
`/api/economics/summary` 가 `{ total_paid_usd, total_realized_pnl_usd, open_positions }` 를 반환하면 NetPnL을 **진짜 realized PnL**로 대체. 단 엔드포인트 없으면 A-2 기본안 유지 (실패 격리).
